## Analysis

### DataWeave処理コスト (c=1, キュー待ちなし)

| シナリオ | 1リクエストコスト | Max RPS (60s) | Echo比 |
|---------|------------------|---------------|--------|
| T1-A Echo | ~3ms | 1,888 | 1x |
| T1-B Small DW | ~3ms | 1,791 (burst) / 674 (stable) | 1x / 2.8x |
| T1-C Heavy DW | ~11ms | 165 | 11.4x |

### スレッドプール効果 (T1-B)

60秒テストでの特徴的な挙動:
- c≤100: CPU 36-48%、RPS ~670 → **スレッドプール制約でCPU未活用**
- c=200: CPU 88%、スレッド122、RPS 1,791 → スレッドプール超えてCPU活用
- c=500: CPU 98%、スレッド131、RPS 1,695 → 飽和
- c=1000: CPU 44%、RPS 434 → 過負荷崩壊

### Mule 4 の過負荷耐性

- c=500/1000 の極端な負荷でもエラーは極めて少ない (0-3.27%)
- レイテンシ増加で吸収する設計 (c=1000 T1-C: avg 6.6秒でもエラー0.7%)
- mule-perf側は常に CPU <20%、スレッド61固定で安定
